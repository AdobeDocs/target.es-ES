## ¿Qué hace? {#section_4ECAACC68723402EB3649033190E1BBC}

| Tipo de actividad | Detalles |
|--- |--- |
| Manual [!UICONTROL Prueba A/B]<P>![icono](/help/main/c-activities/assets/icon_ab.png) | Compara dos o más experiencias para ver cuál mejora más las conversiones durante un periodo de prueba previamente establecido.<P>Para obtener más información, consulte [Pruebas A/B](/help/main/c-activities/t-test-ab/test-ab.md). |
| [!UICONTROL Asignación automática]<P>![Icono de asignación automática](/help/main/c-activities/assets/icon_auto_allocate.png) | Identifica un ganador entre dos o más experiencias y luego redirige el tráfico a la experiencia ganadora para aumentar las conversiones mientras la prueba se ejecuta y aprende.<P>Para obtener más información, consulte [Asignación automática](/help/main/c-activities/automated-traffic-allocation/automated-traffic-allocation.md). |
| [!UICONTROL Segmentación automática]<P>![Icono AT](/help/main/c-activities/assets/icon_auto_target.png) | Utiliza aprendizaje automático avanzado para personalizar el contenido y dirigir las conversiones identificando múltiples experiencias de alto rendimiento definidas por expertos en marketing; y sirve las experiencias más ajustadas a cada visitante en función de su perfil de usuario y el comportamiento pasado de visitantes similares.<P>Para obtener más información, consulte [Establecimiento de destino automático para experiencias personalizadas](/help/main/c-activities/auto-target/auto-target-to-optimize.md). |
| [!UICONTROL Automated Personalization] (AP)<P>![Icono AP](/help/main/c-activities/assets/icon_ap.png) | Utiliza aprendizaje automático avanzado para personalizar el contenido y dirigir las conversiones combinando ofertas o mensajes específicos; a continuación, asigna distintas variaciones de ofertas a los visitantes en función de sus perfiles de cliente.<P>Para obtener más información, consulte [Automated Personalization](/help/main/c-activities/t-automated-personalization/automated-personalization.md). |
| [!UICONTROL Prueba multivariada] (MVT)<P>![Icono MVT](/help/main/c-activities/assets/icon_mvt.png) | Compara combinaciones de ofertas entre elementos de una página para ver qué combinación de ofertas ofrece el mejor rendimiento para una audiencia específica. También identifica qué elemento de la página mejora más las conversiones durante un periodo de prueba previamente establecido.<P>Para obtener más información, consulte [Multivariate Testing](/help/main/c-activities/c-multivariate-testing/multivariate-testing.md). |
| Segmentación de experiencias (XT)<P>![Icono XT](/help/main/c-activities/assets/icon_xt.png) | Ofrece contenido a una audiencia específica en función de un conjunto de reglas y criterios definidos por expertos en marketing.<P>Para obtener más información, consulte [Segmentación de experiencias](/help/main/c-activities/t-experience-target/experience-target.md). |

## ¿Por qué utiliza este tipo de actividad? {#section_46A70DD7CE3448749E635DDF5EAFC131}

| Tipo de actividad | Motivo |
|--- |--- |
| Manual [!UICONTROL Prueba A/B] | Un experimento muy controlado, con mediciones de tráfico, dividido por porcentajes y no por reglas, que le permite analizar los datos de la prueba, comprender mejor a su audiencia y determinar qué experiencia tiene el mejor comportamiento. |
| [!UICONTROL Asignación automática] | Una forma de identificar una experiencia ganadora y ajustar la asignación de tráfico para ofrecer la experiencia ganadora a los visitantes lo más rápido posible, lo que aumenta la probabilidad de conversión. |
| [!UICONTROL Segmentación automática] | Un modo de identificar un ganador entre varias experiencias, para así ofrecer la experiencia más apropiada a cada visitante específico. La segmentación se adapta con el tiempo al variar los intereses de los visitantes, ya que el algoritmo predice una propensión de los visitantes a realizar conversiones en una determinada experiencia y en un determinado momento. |
| [!UICONTROL Automated Personalization] (AP) | Un modo de personalizar un conjunto de ofertas (creadas o predefinidas, en elementos de una sola página o repartidas por múltiples páginas) y proporcionar las combinaciones de ofertas que mejor funcionen para atraer visitantes específicos. |
| [!UICONTROL Multivariate Testing] (MVT) | Una forma de mostrar varias ofertas en varios elementos y, a continuación, probar las experiencias únicas resultantes simultáneamente con un objetivo específico, lo que ayuda a determinar qué variación de elemento es la más exitosa. Una actividad MVT también puede revelar qué elementos tienen el mayor impacto positivo o negativo en la interacción de un visitante. |
| [!UICONTROL Segmentación de experiencias] (XT) | Una forma de dirigir contenido específico a una audiencia específica en función de un conjunto de reglas de asignación definidas. |

## ¿Qué tipo de experto en marketing debe utilizar este tipo de actividad? {#section_A843D663D3E543FFB1A594266B560395}

| Tipo de actividad | El experto en marketing |
|--- |--- |
| Manual [!UICONTROL Prueba A/B] | Sabe manejar estadísticas.<P>Tiene tiempo para esperar hasta el final de la prueba antes de analizar los resultados. |
| [!UICONTROL Asignación automática] | Dispone de poco tiempo.<P>Debe identificar la mejor experiencia y ofrecerla rápidamente.<P>Quiere poder “cotillear” los resultados mientras la prueba se ejecuta. |
| [!UICONTROL Segmentación automática] | Puede elegir entre varias experiencias.<P>Quiere asignar experiencias a visitantes específicos en momentos óptimos en función de sus perfiles dinámicos y cambiantes. |
| [!UICONTROL Automated Personalization] (AP) | Tiene una o más ofertas.<P>Quiere crear combinaciones de ofertas que proporcionen experiencias personalizadas óptimas a visitantes específicos en distintos perfiles y comportamientos únicos. |
| [!UICONTROL Multivariate Testing] (MVT) | Sabe manejar estadísticas.<P>Tiene una o más ofertas.<P>Quiere analizar las tendencias de conversión relativas a las interacciones entre elementos de página. |
| [!UICONTROL Segmentación de experiencias] (XT) | Debe proporcionar una experiencia o contenido específicos a una audiencia específica. |

## Detalles estadísticos {#section_22CF2D07DB054505AB5EC702B99A5BB0}

| Tipo de actividad | Detalles |
|--- |--- |
| Manual [!UICONTROL Prueba A/B] | La prueba compara la experiencia de cada aspirante con una experiencia de control y clasifica el rendimiento de todas ellas, identificando una experiencia ganadora y otra perdedora al compararla con el control. |
| [!UICONTROL Asignación automática] | La prueba produce una garantía estadística de que se obtiene un auténtico ganador y redirige más tráfico hacia las audiencias con mayor probabilidad de conversión con esa experiencia ganadora. |
| [!UICONTROL Segmentación automática] | El mecanismo de optimización identifica la audiencia relevante para cada experiencia mostrando incrementos y disminuciones en el alza a lo largo del tiempo y antes de determinar qué experiencia se ofrece a cada visitante. El mecanismo de optimización se basa en conversiones, segmentos, parámetros y scripts de perfil. A partir de ahí, el mecanismo elige automáticamente qué algoritmo utilizar para generar un alza y una tasa de conversión más altas. |
| [!UICONTROL Automated Personalization] (AP) | El mecanismo de optimización ajusta constantemente las experiencias que se ofrecen a cada visitante en función del comportamiento de los nuevos y los anteriores visitantes similares, al tiempo que compara el rendimiento de una oferta con grupos de control concurrentes. |
| [!UICONTROL Multivariate Testing] (MVT) | La prueba ayuda a desvelar la influencia relativa que determinados elementos tienen en la conversión. |
| [!UICONTROL Segmentación de experiencias] (XT) | El método define reglas que segmentan una experiencia específica o un contenido específico para una audiencia específica. Los clientes pueden realizar actualizaciones en el nivel de experiencia. |

## Ventajas y consideraciones {#section_56C46ABEF7B945DDA0C1E6D714377123}

| Tipo de actividad | Beneficios | Consideraciones |
|--- |--- |--- |
| Manual [!UICONTROL Prueba A/B] | El reposo A/B le ayuda a comprender el rendimiento de cada experiencia, más allá de cuál ofrece el mejor rendimiento. | En un [!UICONTROL Prueba A/B]Sin embargo, si observa los resultados de la prueba antes de alcanzar el tamaño de la muestra, se arriesga a depender de resultados inexactos (¡no puede &quot;asomarse&quot; antes!).<P>Diferente [!UICONTROL Asignación automática]Sin embargo, en una prueba A/B, la distribución del tráfico permanece fija incluso después de reconocer que algunas experiencias se comportan mejor que otras.<P>Para obtener información sobre las prácticas recomendadas para [!UICONTROL Prueba A/B] actividades, consulte [Durante cuánto tiempo se debe ejecutar una prueba A/B](/help/main/c-activities/t-test-ab/sample-size-determination.md) y [Diez dificultades comunes de las pruebas A/B y cómo evitarlas](/help/main/c-activities/t-test-ab/common-ab-testing-pitfalls.md). |
| [!UICONTROL Asignación automática] | [!UICONTROL La asignación automática reduce el coste de una prueba A/B típica porque tiene una mayor tasa de conversión general que una prueba A/B manual. ] La tasa de conversión es mayor porque [!UICONTROL Asignación automática] lleva más tráfico a la experiencia de mayor rendimiento, lo que significa que puede aprovechar las ventajas de esa experiencia ganadora antes del final del período de prueba (puede echar un vistazo). | [!UICONTROL La asignación automática identifica al ganador pero no diferencia entre los perdedores. ] Si debe saber el rendimiento de cada experiencia, es preferible realizar pruebas A/B.<P>El [!UICONTROL Asignación automática] Esta función solo funciona con una configuración de métrica avanzada que es &quot;Aumentar recuento y mantener al usuario en la actividad&quot;. Si no desea contar conversiones repetidas, debe utilizar una prueba A/B en su lugar. |
| [!UICONTROL Segmentación automática] | Con [!UICONTROL Segmentación automática], el aprendizaje automático se aplica a cualquier tipo de experiencia, incluidas las experiencias de varias páginas. Un [!UICONTROL Segmentación automática] La actividad también permite obtener el valor de [!UICONTROL Automated Personalization] al utilizar el conocido flujo de trabajo de pruebas A/B. | Con [!UICONTROL Segmentación automática]Sin embargo, si desea cambiar el contenido de las ofertas con frecuencia o frecuencia, el algoritmo necesita tiempo suficiente después de cada cambio para explotar lo que aprende y entregar ese contenido a los visitantes adecuados. |
| [!UICONTROL Automated Personalization] (AP) | Con [!UICONTROL Automated Personalization], puede recopilar todas las ofertas en un solo lugar y el algoritmo determina la mejor combinación de ofertas. No es necesario especificar o crear experiencias individuales. [!UICONTROL Automated Personalization] utiliza los mismos algoritmos de aprendizaje automático que [!UICONTROL Segmentación automática]. | Cuando combina múltiples ofertas se produce una explosión combinatoria que resulta en la necesidad de aumentar el tráfico de forma significativa. El [!UICONTROL Automated Personalization] algoritmo tiene en cuenta muchos factores, por lo que requiere la mayor cantidad de tráfico.<P>[!UICONTROL Automated Personalization] no se pueden consumir informes en [Analytics for Target](/help/main/c-integrating-target-with-mac/a4t/a4t.md) (A4T). |
| [!UICONTROL Multivariate Testing] (MVT) | Con [!UICONTROL Multivariate Testing], puede probar varios elementos simultáneamente. | A [!UICONTROL Prueba multivariable] lleva tiempo y, debido a las múltiples variables en juego, no produce necesariamente una experiencia ganadora con confianza.<P>A menudo es complicado alcanzar la cantidad de tráfico necesario para completar la prueba. Porque todos [!UICONTROL Prueba multivariable] los experimentos son factoriales completos, demasiados elementos cambiantes a la vez pueden sumar rápidamente muchas combinaciones posibles que deben probarse.<P>Incluso un sitio con tráfico muy elevado puede tener problemas para completar una prueba con más de 25 combinaciones en un tiempo razonable. |
| [!UICONTROL Segmentación de experiencias] (XT) | Con [!UICONTROL Segmentación de experiencias], puede actuar rápidamente con las perspectivas deducidas de los resultados de cualquier actividad.<P>Por ejemplo, si ejecuta una prueba A/B en la que el aspirante no supera al control, pero los resultados indican que un segmento específico de visitantes realizó cuatro veces más conversiones con el aspirante que con el control, puede utilizar [!UICONTROL Segmentación de experiencias] para dirigir la experiencia del aspirante a ese segmento en particular. | [!UICONTROL La segmentación de experiencias no permite controlar el porcentaje de división de una experiencia entre varias audiencias.] |
